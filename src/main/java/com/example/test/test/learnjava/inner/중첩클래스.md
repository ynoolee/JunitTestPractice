# CPU, Car 클래스 와 이에 대한 Main 클래스
이를 테스트하고자 Main 에서 작성한 코드들은 중첩 클래스의 의도와는 다분히 먼 코드들이다. 

중첩 클래스는 하나의 클래스(즉, 이 클래스의 아우터 클래스)에서만 사용되는 클래스를 중첩클래스로 만드는 것이고
여러개의 클래스에서 사용되는 것이라면 탑레벨 클래스로 만드는 것이 맞다. 

Main 에서 직접 중첩 클래스 인스턴스를 생성하여 이를 사용하는 것은 테스트를 위한 것이라고는 하나, 그다지 좋은 방식은 아닌 것 같다. 이런 방식으로 하니 너무 문법적인 측면에만 집착하게 되었다. 

- 단지 static 키워드와 여러가지 접근제한자들에 대한 혼돈과 더불어
- 클래스 내부에 정의된다는 것을 제외하면
- Inner class 는 인스턴스 멤버처럼
- static nested class 는 다른 static member( 클래스 멤버 )

처럼 생각 할 수 있다는 것을 테스트하기 위해 여러 접근을 시도하는 코드를 작성해보는 것으로 한다.
